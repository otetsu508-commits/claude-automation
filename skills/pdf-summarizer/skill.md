# PDF要約スキル

## 概要

PDFファイルを読み込み、その内容を日本語でわかりやすく要約して、指定したフォルダに保存するスキルです。

## 使用方法

ユーザーが以下のような指示をしたときに使用します：

- 「このPDFを要約して [フォルダパス] に保存」
- 「PDFを要約してまとめて [フォルダパス] に保存」
- 「講義資料を要約して [フォルダパス] に保存」
- 「このファイルを要約して（PDFファイルを指定した場合）」
- フォルダパスが省略された場合は、現在のディレクトリまたはデフォルトの場所に保存

## 機能

1. **PDFの読み込み**: OCR処理を使用してPDFファイルからテキストを抽出します
2. **要約の作成**: 抽出された内容を日本語でわかりやすく要約します
3. **ファイル名の作成**: 元のファイル名を基に、わかりやすい要約ファイル名を作成します
4. **指定フォルダへの保存**: ユーザーが指定したフォルダに要約ファイルを保存します

## ワークフロー（OCR優先版）

1. ユーザーがPDFファイルパスと保存先フォルダパスを指定する
   - 例: `[PDFファイルパス] [保存先フォルダパス]`
   - フォルダパスが省略された場合は、現在のディレクトリに保存

2. **OCR処理を実行**
   ```bash
   cd C:\Users\Tetsu\.claude
   python ocr_pdf.py "[PDFファイルパス]"
   ```
   - OCRスクリプトがPDFのページ数を検出
   - 全ページをOCR処理してテキストを抽出
   - 出力ファイル: `[PDFと同じフォルダ]\ocr_output.txt`

3. **OCR出力ファイルを読み込む**
   - `ocr_output.txt`の内容をReadツールで読み込む
   - ユーザーに内容を確認してもらう

4. **内容を要約する**
   - 主なトピックやテーマ
   - 重要なポイント
   - キーワード
   - 適切なマークダウン形式での整理

5. **回数情報を抽出する**
   - OCR出力テキストから「第〇回」のパターンを検索して抽出
   - パターン例: `第⑨回`、`第9回`、`第〇〇回` など
   - 抽出できない場合はフォルダ名を作成しない

6. **保存先フォルダを作成する**
   - 教科フォルダ配下に `要約` フォルダが存在しない場合は作成
   - `要約` フォルダ配下に `第〇回` フォルダを作成
   - コマンド例:
     ```bash
     mkdir -p "c:/Users/Tetsu/OneDrive/大学授業/[教科名]/要約/第〇回"
     ```

7. **要約ファイルを保存する**
   - 保存先: `[教科フォルダ]/要約/第〇回/要約.md`
   - ファイル名は常に `要約.md` とする

8. **OCR出力ファイルを削除する**
   - 要約MDの作成が完了したら、`ocr_output.txt`を削除
   - コマンド: `rm "[PDFと同じフォルダ]/ocr_output.txt"`
   - ユーザーに削除完了を報告する

## 要約のフォーマット

```markdown
# [教科名] - [トピック名]

## 概要
要約の概要を1〜2文で記述

## 主要ポイント
- ポイント1
- ポイント2
- ポイント3

## 詳細内容
### セクション1
内容の説明

### セクション2
内容の説明

## 重要キーワード
- キーワード1
- キーワード2

---
元のファイル: [ファイル名]
作成日: [日付]
```

### フォルダ作成ルール
- 教科フォルダ配下に `要約` フォルダが存在しない場合は自動作成
- `要約` フォルダ配下に `第〇回` フォルダを作成（〇はOCR抽出した回数）
- 回数が抽出できない場合は、フォルダを作成しないか、日付ベースのフォルダ名を使用

## 注意事項

- OCR処理には時間がかかる場合があります。特にページ数が多いPDFは処理に数分以上かかることがあります
- 要約は簡潔かつわかりやすく、初心者でも理解できるように記述
- 専門用語は説明を付けて記述

## 重要：PDF読み込みの厳格なルール（幻覚防止）

### ⚠️ 絶対に守るべきルール

1. **必ずOCR処理でPDFを読み込む**
   - 要約を作成する前に、必ずOCR処理を実行してPDFファイルからテキストを抽出すること
   - 抽出されたテキストのみを使用して要約を作成すること

2. **OCR処理が失敗した場合は正直に報告する**
   - OCRスクリプトが実行できない場合：環境設定を確認するようユーザーに伝える
   - PDFが暗号化されている場合：「このPDFは暗号化されており、読み取れません」と伝える
   - 抽出結果が空白や文字化けの場合：「PDFのテキストが正しく抽出できませんでした」と伝える
   - **いずれの場合も、決して内容を想像・作り上げらないこと**

3. **読み込み内容を確認する**
   - 要約を作成する前に、OCR出力ファイルの内容を一部表示してユーザーに確認を取る
   - 例：「OCRで以下の内容を抽出しました。これを要約しますか？[内容の一部を表示]」

4. **要約は抽出した内容のみに基づく**
   - 抽出したテキストに含まれていない情報を、要約に追加してはならない
   - 一般知識や前提知識で補完してはならない（用語の簡単な説明を除く）

### 📝 ワークフローの修正（OCR優先版）

1. ユーザーがPDFファイルパスを指定
2. **OCR処理を実行（必須）**
   - OCRスクリプトを使ってテキストを抽出
   - 抽出結果を一時ファイルに保存
3. **抽出した内容を確認・表示**
4. **OCR出力から回数情報を抽出**
   - テキスト内の「第〇回」パターンを検索
5. **保存先フォルダを作成**
   - 教科フォルダ/要約/第〇回 の階層を作成
6. ユーザーの確認を得る
7. 抽出した内容のみを要約
8. Markdownファイルを「要約.md」として保存

## 🔍 OCR処理について

### OCR処理の実行

すべてのPDFに対して、最初からOCR処理を実行してテキストを抽出します：

1. **OCRスクリプトの場所**
   - OCRスクリプト: `C:\Users\Tetsu\.claude\ocr_pdf.py`

2. **OCR処理の手順**
   - OCRスクリプトを実行してPDFからテキストを抽出
   - 抽出されたテキストを一時ファイル（`ocr_output.txt`）に保存
   - 一時ファイルの内容を読み込んで要約を作成

3. **コマンド例**
   ```bash
   cd C:\Users\Tetsu\.claude
   python ocr_pdf.py "[PDFファイルパス]"
   ```

### OCR処理に必要な環境

OCR処理を実行するには以下の環境が必要です：

| コンポーネント | 説明 | インストール方法 |
|---------------|------|-----------------|
| Tesseract OCR | OCRエンジン | `winget install --id UB-Mannheim.TesseractOCR` |
| pytesseract | Pythonライブラリ | `pip install pytesseract` |
| PyMuPDF (fitz) | PDF処理ライブラリ | `pip install pymupdf` |
| Pillow | 画像処理ライブラリ | `pip install Pillow` |
| 日本語データ | 日本語OCR用 | `~/tessdata/jpn.traineddata` |

### OCR出力ファイルの場所

OCR処理の出力は以下の場所に保存されます：
- 一時出力: PDFファイルと同じフォルダに `ocr_output.txt` として保存
- 例: `C:\Users\Tetsu\Downloads\ocr_output.txt`（PDFがDownloadsにある場合）

## 大きなPDFの読み込みについて

### 📚 OCR処理の特点

| 項目 | 説明 |
|------|------|
| ページ数制限 | なし（全ページ自動処理） |
| 処理速度 | PDFのページ数に応じて変動 |
| 出力形式 | 全ページのテキストを1つのファイルに結合 |

### 🔄 OCR処理の流れ

OCRスクリプトは自動的にPDFの全ページを処理します：

```
例：60ページのPDFの場合
┌─────────────────────────────────┐
│ OCRスクリプト実行                │
│ → ページ1〜60を順次処理          │
│ → すべてのテキストを抽出          │
└─────────────────────────────────┘
         ↓
    ocr_output.txtに保存
         ↓
    全内容を要約
```

### ⚠️ 注意点

- **処理時間**: ページ数が多いほどOCR処理に時間がかかります
- **進捗表示**: OCRスクリプトが「ページ X/XX を処理中...」と進捗を表示します
- **エラー_handling**: 特定のページでエラーが発生しても、処理は継続されます
