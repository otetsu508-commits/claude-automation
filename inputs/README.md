# コンテンツ入力フォルダ構造

## 概要

このフォルダは5体のAIエージェントコンテンツパイプラインの入力を管理する場所じゃ。

```
inputs/
├── content-calendar.md          # 発信カレンダー
├── README.md                     # このファイル
├── themes/                       # テーマ別のフォルダ
│   ├── 2025-06-22_挫折の3つのパターン/
│   │   └── input.md             # 入力ファイル（テーマ・読者像・背景）
│   └── 2025-06-23_やり直すための3ステップ/
│       └── input.md
└── _archive/                    # 古いファイルのアーカイブ
```

## 入力ファイルの形式

各テーマの `input.md` には以下を記述するのじゃ：

```markdown
# テーマ: [テーマ名]

## 読者像
[誰に向けて書くか]

## プラットフォーム
note記事 + Xスレッド

## 背景（任意）
[なぜこのテーマなのか、補足情報]
```

## 新しい記事を作成する手順

1. `content-calendar.md` で次のテーマを確認
2. `themes/YYYY-MM-DD_テーマ名/` フォルダを作成
3. `input.md` を作成
4. パイプラインを実行
