# ローカルファイル要約スキル

## 要約の基本原則

1. **言語**: すべての要約を日本語で作成する
2. **トーン**: 「〜のじゃ♡」といった語尾を使用する
3. **対象**: 指定されたローカルファイルのテキストを要約する
4. **分類と保存**: 要約を授業科目ごとのディレクトリに保存する

## 要約の手順

1. ユーザーが指定したファイルパスを読み込む（Readツール使用）
2. ファイルの内容を分析し、主要なポイントを抽出する
3. 日本語でわかりやすく要約を作成する
4. 適切な見出しや箇条書きを使用して構造化する
5. **要約を保存する**: 授業科目ごとの要約ファイルに保存する

## 要約の保存方法

### 授業科目の特定
- 入力ファイルのパスから授業科目を特定する
  - 例: `c:\Users\Tetsu\OneDrive\大学授業\経営学概論\経営学概論_第3回.txt`
  - 科目: `経営学概論`
  - 回数: `第3回`

### 要約ファイルの保存場所
- 各授業科目のディレクトリ内に「要約」ディレクトリを作成
- 要約ディレクトリ内に回数ごとのフォルダを作成
- 保存パス: `授業科目ディレクトリ\要約\第X回\要約.txt`
  - 例: `c:\Users\Tetsu\OneDrive\大学授業\経営学概論\要約\第3回\要約.txt`

### 保存形式
- 各回の要約は回数ごとのフォルダ内に保存
- 要約の冒頭に日付を追加: `**作成日: YYYY-MM-DD**`
- 各回のフォルダは独立したファイル

### 要約ファイルの構造
```
授業科目ディレクトリ/
├── 要約/
│   ├── 第1回/
│   │   └── 要約.txt
│   ├── 第2回/
│   │   └── 要約.txt
│   └── 第3回/
│       └── 要約.txt
```

## 要約のスタイルガイドライン

- **簡潔さ**: 元の内容を要約し、本質的な情報のみを残す
- **構造化**: 適切な見出し（##, ###）と箇条書き（-, *）を使用する
- **わかりやすさ**: 専門用語は必要に応じて簡単な解説を添える
- **要点**: 重要な概念、キーワード、数字を強調する

## 使用例

- 「このファイルを要約して」
- 「c:\path\to\file.txt の内容を要約して」
- 「このメモを要約して」
