# 5体のエージェント文章作成スキル

このスキルは5体のエージェントパイプラインを使って、様々な種類の文章を systematic に作成します。

## 対応する文章種類

- **note記事**: カレンダーから今日のテーマを自動取得して作成
- **大学レポート**: テーマ・読者・文字数を収集して作成
- **メール**: 宛先・目的・内容を収集して作成
- **その他**: ユーザーのニーズに応じて対応

## 5体のエージェントの役割

1. **Strategist（戦略担当）**: 切り口・構成を決める
2. **Researcher（リサーチ担当）**: 事実・データ・参考文献を集める
3. **Writer（執筆担当）**: ドラフトを書く
4. **Editor（編集担当）**: 編集・削除・磨く
5. **Publisher（配信担当）**: 最終フォーマットで出力

## パイプラインの実行フロー

### Step 0: まずユーザーに質問（最重要）

スキル発動時に、まずユーザーに以下を質問してください：

```
何を書きますか？
1. note記事
2. レポート
3. メール
4. その他
```

ユーザーの回答に応じて、以下の情報を収集します。

#### note記事の場合

**【重要】まず会社用・個人用を確認**
```
どちらのnote記事を書きますか？
1. 会社のnote
2. 個人のnote
```

- 適切なカレンダーファイルを読み込む：
  - **会社用**: `inputs/content-calendar-company.md` （🔴 会社用）
  - **個人用**: `inputs/content-calendar.md` （🟢 個人用）
- 間違ったタイプのカレンダーを参照しないこと
- 今日の日付（`currentDate`）に対応するテーマを探す
- 入力ファイルが存在しない場合は作成する：
  - フォルダ: `inputs/themes/YYYY-MM-DD_テーマ名/`
  - ファイル: `input.md`
  - 内容: テーマ、読者像、プラットフォーム、コンテキスト、タイプ（会社/個人）

**重要**: note記事でもWeb検索だけでなく論文も引っ張ってきて良い
**Xスレッド**: note記事の場合、Xスレッド（1〜4ツイート）も出力し、最後に「note記事はこちら↓」をつける

#### レポートの場合
以下を質問して情報を収集します：
```
- 題名は何ですか？
- 文字数はどのくらいですか？
- 期限はいつですか？（任意）
```

#### メールの場合
以下を質問して情報を収集します：
```
- 宛先は誰ですか？
- 内容は何ですか？
- 宛先の人物の立場は？（上司、部下、取引先など）
```

#### その他の場合
文章の種類に応じて必要な情報を質問します。

### Step 2: Strategist（切り口を決める）

content-strategist サブエージェントを起動します。

**入力**:
- テーマ
- 読者像
- プラットフォーム（note、レポート形式、メールなど）
- 文字数
- コンテキスト

**出力**:
- 角度（1行）
- フック（1行、8文字以内）
- ブリーフ（構成・字数・トーン）
- 書かないこと（3点）

### Step 3: Researcher（事実を集める）

content-researcher サブエージェントを起動します。

**文章種類による使い分け**:
- **レポート**: 参考文献を載せる、論文から事題を持ってくる
- **note記事**: Web検索・一次情報収集
- **メール**: 必要に応じて事実確認（簡略化可）

### Step 4: Writer（ドラフトを書く）

content-writer サブエージェントを起動します。

**入力**:
- Strategist のブリーフ
- Researcher のリサーチパック

**出力**:
- フルドラフト

### Step 5: Editor（編集する）

content-editor サブエージェントを起動します。

**タスク**:
- 30%削る
- フックを磨く
- 締めを引用したくなる形に書き直す

**出力**:
- 編集済みドラフト
- 変更ログ

### Step 6: Publisher（出力する）

content-publisher サブエージェントを起動します。

**タスク**:
- 最終フォーマットで出力
- 所定の場所に保存

**保存場所**:
- note記事（会社用）: `outputs/company/YYYY-MM-DD_テーマ名/`
- note記事（個人用）: `outputs/personal/YYYY-MM-DD_テーマ名/`
- レポート: `outputs/reports/`
- メール: `outputs/emails/`
- その他: `outputs/others/`

**【絶対遵守】会社用と個人用の出力先は絶対に混ぜないでください**
- 会社のnoteは `outputs/company/` のみ
- 個人のnoteは `outputs/personal/` のみ
- 間違っても混ぜるとコンテンツがごちゃ混ぜになり大変です

## 出力ファイル形式

### note記事の場合（会社用）
```
outputs/company/YYYY-MM-DD_テーマ名/
├── 01-angle.md      # Strategist の出力
├── 02-research.md   # Researcher の出力
├── 03-draft.md      # Writer の出力
├── 04-edited.md     # Editor の出力
└── 05-final.md      # Publisher の出力（note記事 ＋ Xスレッド）
```

### note記事の場合（個人用）
```
outputs/personal/YYYY-MM-DD_テーマ名/
├── 01-angle.md      # Strategist の出力
├── 02-research.md   # Researcher の出力
├── 03-draft.md      # Writer の出力
├── 04-edited.md     # Editor の出力
└── 05-final.md      # Publisher の出力（note記事 ＋ Xスレッド）
```

**note記事の最終ファイルには以下を含める**:
- note記事本文（タイトル・タグ・本文）
- Xスレッド（1〜4ツイート、最後に「note記事はこちら↓」）

### レポートの場合
```
outputs/reports/
└── YYYY-MM-DD_テーマ名/
    ├── 01-angle.md
    ├── 02-research.md
    ├── 03-draft.md
    ├── 04-edited.md
    └── 05-final.md
```

### メールの場合
```
outputs/emails/
└── YYYY-MM-DD_宛先_目的/
    ├── 01-angle.md
    ├── 02-research.md（必要な場合）
    ├── 03-draft.md
    ├── 04-edited.md
    └── 05-final.md（メール形式）
```

## ハンドオフプロトコル

各エージェント間の引き継ぎには、以下のテンプレートを使用します：

```markdown
# ハンドオフ: [From] → [To]

## 1. シーケンス上の位置
現在: 5 段階中の N/5 完了、[To] 開始

## 2. 前の agent が完了したこと
- （完了したことのサマリー）

## 3. 主要決定事項（[To] が引き継ぐべき判断）
- （主要決定事項）

## 4. 未決の問い（[To] に判断を委ねる）
- （未決の問い）

## 5. [To] への申し送り
- タスク: {具体的なタスク}
- 重視: {重視する点}
- 回避: {回避する点}
```

## 使用例

### note記事を作成する場合
```
ユーザー: 「今日のnote記事を書いて」
→ 会社用・個人用を確認
→ カレンダーから今日のテーマを取得
→ 5体のパイプラインを実行
→ outputs/company/ または outputs/personal/ に最終ファイルを出力
```

### レポートを作成する場合
```
ユーザー: 「レポートを書いて。テーマは〇〇で...」
→ テーマ・読者・文字数を確認
→ 5体のパイプラインを実行（Researcherは参考文献重視）
→ outputs/reports/ に最終ファイルを出力
```

### メールを作成する場合
```
ユーザー: 「メールを作って。宛先は...」
→ 宛先・目的・内容を確認
→ 5体のパイプラインを実行（Researcherは簡略化）
→ outputs/emails/ に最終ファイルを出力
```

## 注意点

- **まず最初にユーザーに質問すること**: 何を書くのか（note記事/レポート/メール/その他）を確認
- **note記事の場合は会社用・個人用を必ず確認**: 間違ったタイプを出力しない
- **会社用・個人用の出力先は絶対に混ぜない**:
  - 会社用: `outputs/company/` のみ
  - 個人用: `outputs/personal/` のみ
- **各エージェントは順番通りに実行**: Strategist → Researcher → Writer → Editor → Publisher
- **文章種類によって Researcher の扱いを変える**: レポートは参考文献重視、メールは簡略化
- **note記事でも論文を使用**: Web検索だけでなく論文も引っ張ってきて良い
- **カレンダー参照時は `currentDate` を使用**: 今日の日付でテーマを特定
- **すべての中間ファイルも保存**: 後で確認できるように
