# Article Scraper

このスキルはURLを指定して記事を抽出し、Markdownファイルとして保存します。

## いつ使うスキルか

このスキルはユーザーが次のようなことを求めたときに使用します：

- 「このURLの記事を保存して」
- 「URLから記事を抽出してMarkdownにして」
- 「記事をスクラップして要約して」
- 「このメモをようやくして」

## 実行手順

### 手順 1: ユーザーからURLを取得

ユーザーがURLを指定しているか確認します。URLがない場合は尋ねてください。

### 手順 2: 記事コンテンツを取得

MCP `mcp__web_reader__webReader` ツールを使ってURLのコンテンツを取得します。

```python
mcp__web_reader__webReader(
    url="<ユーザー指定のURL>",
    return_format="markdown"
)
```

### 手順 3: フロントマターを生成

取得したコンテンツから以下の情報を生成します：

| フィールド | 説明 |
|-----------|------|
| title | 記事のタイトル（HTMLのtitleタグや見出しから推測。見つからない場合は「記事タイトル不明」） |
| url | 指定されたURL |
| tag | 記事内容からAI推測（カンマ区切り、例: プログラミング,AI,技術） |
| created date | 作成日（YYYY-MM-DD形式） |
| summary | 記事の要約（150文字以内） |

### 手順 4: 記事本文を作成

以下の構成で記事本文を作成します：

```markdown
> [!NOTE] 引用
> 引用元: <URL>

# 概要
<summaryと同じ内容（150文字以内）>

# 詳細
<3000文字以内の要約本文>

# まとめ
<記事のまとめ>
```

### 手順 5: ファイルに保存

`docs/articles/` ディレクトリに `{title}.md` として保存します。

- ディレクトリが存在しない場合は作成してください
- ファイル名はタイトルから特殊文字を除去したものにしてください

## 例外処理

| エラー | 対応 |
|--------|------|
| URLが無効 | エラーメッセージを表示して、有効なURLを再度尋ねる |
| コンテンツ取得失敗 | エラーメッセージを表示して、原因を説明 |
| タイトルが見つからない | 「記事タイトル不明」として処理 |

## 使用例

ユーザー入力：
```
このURLの記事を保存して: https://example.com/interesting-article
```

実行：
1. URLのコンテンツを取得
2. タイトル、タグ、要約を生成
3. `docs/articles/interesting-article.md` に保存
